01 | Why Now

為什麼是現在

AI Agent 的信任缺口，不能只靠更好的回答品質解決。

當 AI Agent 開始代替人決定、查詢、下單、送件或簽署時，風險不再只是模型答錯，而是可能發生越權操作、個資外洩、責任不明、授權無法撤銷，或無法證明某個行動由誰授意。

它代表誰？ 被授權做什麼？ 授權何時失效？ 哪些動作需人類確認？ 出事後如何追溯與問責？

04 | Tracks

四大賽道

One backbone, four scenarios. 每個作品都需回應一個具體場景中的可信問題。每隊只能選擇一個賽道，不可跨賽道投件。

供應鏈與貿易金融

場景痛點
供應商資格、訂單融資、ESG／碳資料可信度。
核心可信問題
Agent 是否代表採購人／法人？能否讀取供應商憑證、草擬訂單，但禁止自行付款？
示例 POC
供應商資格 VC、訂單即融資證明、ESG／碳盤查選擇性揭露、採購 Agent 授權。

電商與第三方支付

場景痛點
高頻代購、支付授權、反詐騙、資格驗證。
核心可信問題
Agent 是否僅能比價與建立付款意向？何時必須由本人確認？
示例 POC
Agent 代購授權、商家可信驗證、年齡／資格選擇性揭露、付款前 human approval。

醫療保險

場景痛點
高敏感個資、理賠、同意與核保稽核。
核心可信問題
Agent 可以讀取哪些醫療資料？是否有有效 consent？送件前如何確認？
示例 POC
理賠資料可信交換、同意 VC、隱私保護下聯合分析、AI 核保 action log。

政府服務

場景痛點
公部門憑證、跨部會、代辦與國際互認。
核心可信問題
Agent 可以代填、代查或代送到何種程度？哪些動作需本人確認？
示例 POC
公部門憑證升級 VC、Agent 代辦申請、資格驗證、跨境互認概念。

05 | Trust Foundation

共同主軸「信任地基」

所有賽道都需處理 identity · authorization · verification · privacy · security · accountability。

identity authorization verification privacy security accountability

核心必備

Principal

代表誰？Agent 代表哪一個人、團隊、法人或公部門單位。

Authorization

被授權做什麼？誰授權、允許做什麼、禁止做什麼。

Tool / Action

能呼叫什麼工具？每個工具需要哪些權限與邊界。

Policy Gate

高風險動作如何被擋？付款、送件、簽署或讀取敏感資料需明確把關。

Audit Log

如何追溯行動？記錄 Agent 的行動、決策與授權依據。

Expiry / Revocation

何時失效或撤銷？授權過期或撤銷後，Agent 必須停止執行。

補充交付

Goal / Task

Agent 的具體任務與完成條件是什麼？是否不是一般問答 chatbot？

Governance Gap Memo

以一頁 memo 說明台灣落地所需的法規、標準、責任或資料治理缺口。

核心必備 6 項是評審會檢查的最低信任規格；Goal / Task 與 Governance Gap Memo 則用來說明任務範圍與台灣落地治理缺口。

單純 chatbot 包裝不足以拿高分。作品應展示 Agent 在明確信任限制下完成真實任務，且能被驗證、限制、追溯。

06 | Eligibility & Rules

參賽資格與規則

公開版只顯示已確認摘要；細節以正式簡章為準。

參賽對象

  • 學生、青年開發者、業界開發者。
  • AI／資料、資安、區塊鏈、產品、設計或專案管理人才。
  • 供應鏈、支付、醫療保險、政府服務等領域參與者。
  • 鼓勵跨專業組隊，但所有成員必須在同一完整隊伍內報名。

隊伍與出席

  • 每隊 3–5 人，每位參賽者限加入一隊。

出席與交付

  • 兩場賽前工作坊限入選隊伍參與，不開放旁聽。工作坊不是強制全體成員參加，但強烈建議每隊至少派代表參與，以完成工具理解、賽道痛點討論與 POC 收斂。
  • Demo Day 至少需有隊伍成員進行中文簡報。
  • 須完成原型、Demo、簡報、README 與治理／信任設計說明；確切交件格式以正式簡章為準。

安全與資料

  • 僅使用具合法授權的模型、資料、工具及第三方服務。
  • 真實醫療、金融、政府或其他敏感資料不得使用。
  • 敏感場景應使用匿名、合成或測試資料。
  • 智慧財產權、影像授權、獎金稅務、夜間與未成年規範以正式簡章為準。

12 | FAQ

常見問答

報名前可先確認資格、交付與規則。

A. 報名與隊伍

這場活動是免費的嗎？

完全免費。入選隊伍可獲得 Builder Kit（API credits／開發資源），活動期間提供餐食。

誰可以參加？

對 AI、軟體開發、設計、產品或相關領域有興趣的學生、社會人士與業界人士皆可報名。

每隊幾人？

每隊 3–5 人，且每位參賽者僅能參加一隊。

可以個人報名嗎？

本屆僅收完整隊伍報名，請先自行組隊（3–5 人）再報名。

怎麼報名？

於報名期間填寫線上報名表：7/8 開放、8/5 截止。

B. 時程與賽道

重要時程有哪些？

7/8 開放報名、7/20 線上說明會、8/5 截止、8/7 公告入選、8/15 ＆ 8/22 賽前工作坊、8/29–8/30 黑客松、8/31 Demo Day。

要選賽道嗎？

報名時選擇一個主賽道：供應鏈與貿易金融、電商與第三方支付、醫療保險、政府服務。

技術或框架有限制嗎？

技術自由，但作品須能展示「可信 AI Agent 最低規格」六項，且僅能使用合法授權之模型、資料與工具。

「可信 AI Agent 最低規格」是什麼？

代表誰（Principal）、被授權做什麼（Authorization）、可用的工具與動作（Tool/Action）、政策關卡（Policy Gate）、稽核軌跡（Audit Log）、失效與撤銷（Expiry/Revocation）。

需要自備設備嗎？

請自備筆電與開發所需裝置；現場提供電力、網路與基本設備。

C. 入選、交付與評分

入選後要做什麼準備？

入選隊伍需出席 8/15、8/22 兩場賽前工作坊，並準備 8/29–8/31 全程參與。

一定要參加工作坊嗎？

是，兩場工作坊（8/15、8/22）僅限入選隊伍、不開放旁聽，兩天皆提供餐食。

要交什麼？

程式碼儲存庫（GitHub）、簡報、Demo 連結或影片、一頁治理／信任設計說明、README。

交件截止什麼時候？

8/30（日）23:59 前完成交件，缺件可能影響評分或資格。

怎麼評分？

初選：題目相關性 50%、可行性 30%、團隊能力 20%；決選：信任架構 40%、技術執行 35%、場景適配 25%。

D. 法務、場務與聯絡

獎項有哪些？

現金獎金池 USD 12,000+（將隨贊助加碼）：冠軍 1 隊 USD 5,000、亞軍 2 隊各 USD 2,000、季軍 3 隊各 USD 1,000；另有四大賽道獎，入選 20 隊皆獲 Builder Kit。

提供過夜嗎？

8/29→8/30 提供過夜，自由參加，可選擇不過夜或當日往返；過夜需於報名時表明，並採名額與場地安全管理。

未成年可以參加嗎？

未滿 18 歲須經法定代理人同意；如需過夜，須另提供家長同意書並指定緊急聯絡人。

智慧財產權與個資怎麼處理？

作品智財原則上歸該隊；主辦與合作夥伴得為紀錄、成果發表與宣傳使用作品名稱、簡介與展示內容。個資僅用於本活動之聯絡、審查、保險與行政，並依個資法令辦理；活動將攝錄影並可能用於宣傳。
