01 | Why Now

為什麼是現在

AI Agent 的信任缺口，不能只靠更好的回答品質解決。

當 AI Agent 開始代替人決定、查詢、下單、送件或簽署時，風險不再只是模型答錯，而是可能發生越權操作、個資外洩、責任不明、授權無法撤銷，或無法證明某個行動由誰授意。

它代表誰？ 被授權做什麼？ 授權何時失效？ 哪些動作需人類確認？ 出事後如何追溯與問責？

04 | Tracks

四大賽道

One backbone, four scenarios. 每個作品都需回應具體場景中的可信問題。

供應鏈與貿易金融

場景痛點
供應商資格、訂單融資、ESG／碳資料可信度。
核心可信問題
Agent 是否代表採購人／法人？能否讀取供應商憑證、草擬訂單，但禁止自行付款？
示例 POC
供應商資格 VC、訂單即融資證明、ESG／碳盤查選擇性揭露、採購 Agent 授權。

電商與第三方支付

場景痛點
高頻代購、支付授權、反詐騙、資格驗證。
核心可信問題
Agent 是否僅能比價與建立付款意向？何時必須由本人確認？
示例 POC
Agent 代購授權、商家可信驗證、年齡／資格選擇性揭露、付款前 human approval。

醫療保險

場景痛點
高敏感個資、理賠、同意與核保稽核。
核心可信問題
Agent 可以讀取哪些醫療資料？是否有有效 consent？送件前如何確認？
示例 POC
理賠資料可信交換、同意 VC、隱私保護下聯合分析、AI 核保 action log。

政府服務

場景痛點
公部門憑證、跨部會、代辦與國際互認。
核心可信問題
Agent 可以代填、代查或代送到何種程度？哪些動作需本人確認？
示例 POC
公部門憑證升級 VC、Agent 代辦申請、資格驗證、跨境互認概念。

05 | Trust Foundation

共同主軸「信任地基」

所有賽道都需處理 identity · authorization · verification · privacy · security · accountability。

identity authorization verification privacy security accountability

Principal

代表誰？Agent 代表哪一個人、團隊、法人或公部門單位。

Goal / Task

Agent 的具體任務與完成條件是什麼？是否不是一般問答 chatbot？

Authorization

被授權做什麼？誰授權、允許做什麼、禁止做什麼。

Tool / Action

能呼叫什麼工具？每個工具需要哪些權限與邊界。

Policy Gate

高風險動作如何被擋？付款、送件、簽署或讀取敏感資料需明確把關。

Audit Log

如何追溯行動？記錄 Agent 的行動、決策與授權依據。

Expiry / Revocation

何時失效或撤銷？授權過期或撤銷後，Agent 必須停止執行。

Governance Gap

以一頁 memo 說明台灣落地所需的法規、標準、責任或資料治理缺口。

單純 chatbot 包裝不足以拿高分。作品應展示 Agent 在明確信任限制下完成真實任務，且能被驗證、限制、追溯。

06 | Eligibility & Rules

參賽資格與規則

鼓勵跨校、跨公司、跨技術與領域組隊。

參賽對象

  • 學生、青年開發者與業界開發者。
  • AI／資料、資安、區塊鏈、產品、設計或專案管理人才。
  • 供應鏈、支付、醫療保險、政府服務等領域參與者。

隊伍與出席

  • 每隊 2-5 人，每位參賽者限加入一隊。
  • 可個人報名，再由主辦方協助媒合。
  • 入選隊伍需完整參與 8/29-31 主要流程。

交付與注意事項

  • 需完成 demo、中文簡報與一頁治理缺口 memo。
  • 僅使用具合法授權的模型、資料、工具與第三方服務。
  • 敏感資料以匿名、合成或測試資料替代，並遵守 sponsor 使用規範。

14 | FAQ

常見問答

報名前可先確認資格、交付與規則。

A. 報名與組隊

誰可以參加？

學生、青年開發者、業界開發者、設計／產品人才，以及供應鏈、支付、醫療保險、政府服務等四大賽道相關領域參與者皆可報名。

需要具備哪些背景？

不限定單一背景。建議隊伍具備開發、AI／資料、資安、產品、設計、專案管理或領域知識。

可以個人報名嗎？如何媒合？

可以。個人可先報名，主辦方將依技能、賽道與可投入時段協助媒合。

每隊幾人？每位參賽者可否加入多隊？

每隊 2–5 人；每位參賽者限加入一隊。

可以跨校、跨公司或跨領域組隊嗎？

可以，也鼓勵跨校、跨公司、跨技術與領域組隊。

每隊是否只能選一個賽道？跨賽道作品如何評審？

每隊需指定一個主賽道；跨賽道作品仍以主賽道作為主要評審場景。

入選後是否必須全程出席？

入選隊伍需完整參與 8/29–31 主要流程與 Demo Day。

初選如何評分？是否有候補機制？

初選依題目相關性 50%、可行性 30%、團隊能力 20% 評分；候補機制以正式簡章為準。

報名何時開放？

報名即將開放，正式時間以活動公告為準。

B. 開發與技術

什麼才算可信 AI Agent？一般 chatbot 可以嗎？

作品需展示 Agent 代表誰、被授權做什麼、能呼叫什麼工具、如何被 policy gate 限制、如何追溯與撤銷；單純 chatbot wrapper 不足以拿高分。

可以使用哪些模型、程式語言與 Agent framework？

技術棧自由，可使用任一模型、程式語言、Agent framework、雲端或本地端環境。

是否一定要使用 VC、DID、KERI、vLEI 或政府數位憑證皮夾？

不強制。這些都是可選技術或加分方向，不是所有隊伍的硬性技術棧。

可以使用既有程式碼與開源套件嗎？

可以，但需揭露並遵守其授權條款。

可以使用真實醫療、金融或政府資料嗎？

不可以。敏感資料應使用匿名、合成或測試資料。

哪些高風險動作需 human approval？

付款、送件、簽署、讀取敏感資料或造成法律／業務效果的動作，應設計 human approval 或阻擋機制。

C. 交件與評分

必須提交哪些內容？

至少包含可運作原型、現場 Demo、中文簡報、程式碼／技術文件與一頁台灣落地治理缺口 memo。

Demo Day 使用什麼語言？

Demo Day 以中文簡報與現場成果展示為主；技術名稱、程式碼、專有名詞可使用英文。

初選 50/30/20 的評分方式為何？

題目相關性 50%、可行性 30%、團隊能力 20%。

決選 40/35/25 的評分方式為何？

信任架構 40%、技術執行 35%、場景適配 25%。

完賽與取得 Completion VC 的條件為何？

需到場參與、完成交件、完成 Demo Day 展示；細節以正式簡章為準。

D. 資源與獎項

Builder Kit / AI credits 如何提供？

提供給 20 隊入選隊伍，以隊伍為單位並可分階段發放。

兩場工作坊是否必須參加？

入選隊伍應參與必要說明、兩場工作坊與技術彩排；實際要求以正式簡章為準。

現金獎如何分配？

主現金獎池 USD 12,000：冠軍 1 隊 x USD 5,000、亞軍 2 隊 x USD 2,000、季軍 3 隊 x USD 1,000。

主獎與 Best Track Award 可以重疊嗎？

可以；同一隊原則最多一個主現金獎加一個賽道獎，特別獎另計。

Mac mini／DGX Spark 是否確定？

Mac mini 為四大賽道獎目標硬體，DGX Spark 為 sponsor-backed 選配特別獎；皆以正式贊助公告為準。

E. 法務、場務與聯絡

智慧財產權、影像授權、獎金稅務如何處理？

正式簡章另行公告，不在本頁自行創造條款。

活動地點與日期？

2026/08/29–8/30 黑客松；2026/08/31 Demo Day。地點為 N24 台北方舟。

需要自行攜帶哪些設備？

請自備開發所需筆電與周邊；其他現場設備需求以行前通知為準。

夜間活動、餐飲、網路與休息安排何時公告？

以正式簡章與入選後行前通知為準。

有其他問題如何聯絡？

請聯絡 Taka｜taka@chain.tw。
