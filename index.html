01 | Why Now

為什麼是現在

AI Agent 的信任缺口，不能只靠更好的回答品質解決。

當 AI Agent 開始代替人決定、查詢、下單、送件或簽署時，風險不再只是模型答錯，而是可能發生越權操作、個資外洩、責任不明、授權無法撤銷，或無法證明某個行動由誰授意。

它代表誰？ 被授權做什麼？ 授權何時失效？ 哪些動作需人類確認？ 出事後如何追溯與問責？

04 | Tracks

四大賽道

One backbone, four scenarios. 每個作品都需回應一個具體場景中的可信問題。每隊只能選擇一個賽道，不可跨賽道投件。

供應鏈與貿易金融

場景痛點
供應商資格、訂單融資、ESG／碳資料可信度。
核心可信問題
Agent 是否代表採購人／法人？能否讀取供應商憑證、草擬訂單，但禁止自行付款？
示例 POC
供應商資格 VC、訂單即融資證明、ESG／碳盤查選擇性揭露、採購 Agent 授權。

電商與第三方支付

場景痛點
高頻代購、支付授權、反詐騙、資格驗證。
核心可信問題
Agent 是否僅能比價與建立付款意向？何時必須由本人確認？
示例 POC
Agent 代購授權、商家可信驗證、年齡／資格選擇性揭露、付款前 human approval。

醫療保險

場景痛點
高敏感個資、理賠、同意與核保稽核。
核心可信問題
Agent 可以讀取哪些醫療資料？是否有有效 consent？送件前如何確認？
示例 POC
理賠資料可信交換、同意 VC、隱私保護下聯合分析、AI 核保 action log。

政府服務

場景痛點
公部門憑證、跨部會、代辦與國際互認。
核心可信問題
Agent 可以代填、代查或代送到何種程度？哪些動作需本人確認？
示例 POC
公部門憑證升級 VC、Agent 代辦申請、資格驗證、跨境互認概念。

05 | Trust Foundation

共同主軸「信任地基」

所有賽道都需處理 identity · authorization · verification · privacy · security · accountability。

identity authorization verification privacy security accountability

Principal

代表誰？Agent 代表哪一個人、團隊、法人或公部門單位。

Goal / Task

Agent 的具體任務與完成條件是什麼？是否不是一般問答 chatbot？

Authorization

被授權做什麼？誰授權、允許做什麼、禁止做什麼。

Tool / Action

能呼叫什麼工具？每個工具需要哪些權限與邊界。

Policy Gate

高風險動作如何被擋？付款、送件、簽署或讀取敏感資料需明確把關。

Audit Log

如何追溯行動？記錄 Agent 的行動、決策與授權依據。

Expiry / Revocation

何時失效或撤銷？授權過期或撤銷後，Agent 必須停止執行。

Governance Gap

以一頁 memo 說明台灣落地所需的法規、標準、責任或資料治理缺口。

單純 chatbot 包裝不足以拿高分。作品應展示 Agent 在明確信任限制下完成真實任務，且能被驗證、限制、追溯。

06 | Eligibility & Rules

參賽資格與規則

公開版只顯示已確認摘要；細節以正式簡章為準。

參賽對象

  • 學生、青年開發者、業界開發者。
  • AI／資料、資安、區塊鏈、產品、設計或專案管理人才。
  • 供應鏈、支付、醫療保險、政府服務等領域參與者。
  • 鼓勵跨專業組隊，但所有成員必須在同一完整隊伍內報名。

隊伍與出席

  • 每隊 3-5 人，每位參賽者限加入一隊。

出席與交付

  • 入選隊伍應參與必要說明、兩場工作坊、技術彩排及活動主要流程；實際最低出席要求以正式簡章為準。
  • Demo Day 至少需有隊伍成員進行中文簡報。
  • 須完成原型、Demo、簡報、README 與治理／信任設計說明；確切交件格式以正式簡章為準。

安全與資料

  • 僅使用具合法授權的模型、資料、工具及第三方服務。
  • 真實醫療、金融、政府或其他敏感資料不得使用。
  • 敏感場景應使用匿名、合成或測試資料。
  • 智慧財產權、影像授權、獎金稅務、夜間與未成年規範以正式簡章為準。

12 | FAQ

常見問答

報名前可先確認資格、交付與規則。

A. 報名與隊伍

每隊幾人？

每隊 3-5 人，由 Team Lead 代表完整隊伍提交報名。

是否接受個人報名或主辦方媒合？

目前只接受完整隊伍報名，不提供個人報名或主辦方組隊媒合。

每位參賽者可以加入多隊嗎？

不可以，每位參賽者限加入一隊。

可以跨賽道嗎？

不可以。每隊只能選擇一個賽道，同一作品不得跨賽道投件或同時參與多個賽道評審。

報名何時開放？

報名已於 2026 年 7 月 8 日正式開放，報名截止時間為 2026 年 8 月 5 日。請由本頁報名按鈕前往表單填寫。

B. 初選與作品

初選如何評分？是否有候補機制？

初選依題目相關性 50%、可行性 30%、團隊能力 20% 評分；候補機制以正式簡章為準。

什麼才算可信 AI Agent？一般 chatbot 可以嗎？

作品需展示 Agent 代表誰、被授權做什麼、能呼叫什麼工具、如何被 policy gate 限制、如何追溯與撤銷；單純 chatbot wrapper 不足以拿高分。

可以使用哪些模型、程式語言與 Agent framework？

技術棧自由，可使用任一模型、程式語言、Agent framework、雲端或本地端環境。

是否一定要使用 VC、DID、KERI、vLEI 或政府數位憑證皮夾？

不強制。這些都是可選技術或加分方向，不是所有隊伍的硬性技術棧。

可以使用既有程式碼與開源套件嗎？

可以，但需揭露並遵守其授權條款。

可以使用真實醫療、金融或政府資料嗎？

不可以。敏感資料應使用匿名、合成或測試資料。

哪些高風險動作需 human approval？

付款、送件、簽署、讀取敏感資料或造成法律／業務效果的動作，應設計 human approval 或阻擋機制。

C. 工作坊與獎項

兩場工作坊是否必須參加？

入選隊伍應參與必要說明、兩場工作坊與技術彩排；實際要求以正式簡章為準。

獎金如何安排？

現金獎金池 USD 12,000，持續加碼中。

D. 法務、場務與聯絡

智慧財產權、影像授權與獎金稅務如何處理？

正式簡章另行公告，本頁不自行創造或擴張法律條款。

活動地點與日期？

2026/08/29–8/30 為 Hackathon，2026/08/31 為 Demo Day；地點為 N24 台北方舟。

需要自行攜帶哪些設備？

請自備開發所需筆電與周邊；其他現場設備需求以行前通知為準。

夜間活動、餐飲、網路與休息安排何時公告？

以正式簡章與入選後行前通知為準。
